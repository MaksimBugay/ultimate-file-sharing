Secure FileShare

Provide public secure internet access to local files on your PC in two clicks

Get started:

  1. Press ‘add files’ button
  2. Enter strong password if you want to restrict access to your files
  3. Open ‘select files’ dialog by pressing  button
  4. Optionally pack your files into ZIP archive to speedup download and save network traffic. Provide desired archive name (first selected file name will be used as an archive name otherwise)
  5. Once files are selected they will be added into internal browser storage and public access link will be copied into keyboard buffer. You can past that link into any place: social media comments, messengers etc.
  6. Press button in credentials column to copy credentials and share them via secure reliable channel
  7. Use manger page to administrate your shared files

Ten reasons to start using:

  1. No need to setup SFTP server or any similar software on your machine
  2. No need in public IP address
  3. No need to open any ports on your machine
  4. Your files are always hosted on your PC and never copied to any cloud
  5. No registration required
  6. It is completely free
  7. No file owner data disclosure
  8. Exceptional level of protection against unauthorized access for files shared with strong secure password
  9. Short fully anonymous public links that can be shared via any channel (messengers, social networks etc.)
  10. Only secure https/wss protocols are used for data transfer


 Why our password protection is almost impossible to huck:

  1. Receiver (for instance friend of you) will be redirected to pre-download page after attempt to open short public link

  1. Now He should enter credentials that you previously shared with him via a secure, reliable channel. Never send credentials along with download link!!!
  2. Download request with signature be created and sent via https POST channel to file owner. Signature is created based on provided workspace ID and password with strong and secure cryptography algorithm
  3. So actual credentials are never sent via network or stored on any cloud server
  4. File owner verify download request signature and initiate upload if signature is correct
  5. For better security download request has very short ttl(time to live)
  6. Do not use “Expose workspace ID’’ option if your goal is an ultimate security and protection against brute force attacks
